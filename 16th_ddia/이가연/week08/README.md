## 선형성

하나의 DB 복제본만 있다면 항상 read마다 최신 값을 보장해준다. → 강한 일관성

그러나 복제본이 2개가 된다면 동기화 시점 차이 때문에 최신값을 보장하지 못한다. 조회 시점에 따라 동기화가 됐을 수도 있고 안됐을 수도 있기 때문이다. → 비선형 시스템

선형성이란 데이터 복사본이 하나만 있는 것처럼, 그래서 그 데이터를 대상으로 수행하는 모든 연산은 원자적인 것처럼 보이게 만드는 것, 즉, 목적은 **최신성 보장. (**직렬성과는 다르다.)

### 잠금과 리더 선출

단일 리더 복제 시스템에서 리더는 정말 하나여야 한다. 리더는 모든 노드가 시작할 때 락 획득을 시도하고 가장 먼저 성공한 노드가 리더가 되는데, 이 락 소유는 선형적이어야 하고 모든 노드가 락 소유에 합의해야 한다.

여기서 락 소유가 선형적이어야 한다는 얘기는 **누가 락을 가졌는가?** 라는 질문에 대해 **모든 노드가 같은 답**을 하는 것을 의미한다.

### 제약 조건과 유일성 보장

유니크 제약 조건이 유효하려면 선형성이 필요하다. 이 연산은 CAS와 비슷한데 서로 읽는 값이 다르면 중복이 들어갈 수 있기 때문.

## 선형적 시스템 구현하기

선형성은 근본적으로 “데이터 복사본이 하나만 있는 것처럼 동작하고 그 데이터에 실행되는 모든 연산은 원자적” → 그러나 실제로 복사본이 1개라면 결함을 견뎌낼 수 없다 → 내결함성을 지니기 위해 복제는 필수불가결

복제 방법들

- 단일 리더 복제 (선형적일 확률 있음, 스냅샷 격리 등을 사용한다면 선형적이지 않을 수도 있다.)
- 합의 알고리즘 (선형적)
- 다중 리더 복제 (비선형적)
- 리더 없는 복제 (논란은 있으나 대부분 비선형적, 최종 쓰기 승리도 시계 스큐 문제 때문에 선형성의 가능성을 망친다.)

### 선형성과 정족수

다이나모 모델에서 엄격한 정족수를 사용한 읽기 쓰기는 선형적으로 보이지만 실제로는 네트워크 지연 변동이 심하면 경쟁 조건이 생길 수 있다. (정족수 : 안건을 가결하는데 필요한 최소 인원 )

- 다이나모 모델의 엄격한 정족수(strict quorum)
    
    ```json
    N = 복제본 수
    W = 쓰기 성공에 필요한 노드 수
    R = 읽기에 필요한 노드 수
    ```
    
    → R + W > N 이면 엄격한 정족수를 만족한다고 한다.
